    3. Atomic Last-Admin Protections¶

    Files changed:

    • sidecar/internal/auth/auth.go
    • sidecar/internal/api/auth_handlers.go
    • sidecar/internal/api/coverage_phase2_test.go

    Behavior now:

    • Added auth.ErrLastAdmin.
    • Added auth.DeleteUserPreservingAdmin.
    • Added auth.UpdateUserRolePreservingAdmin.
    • Both begin a transaction and LOCK TABLE sage.users IN EXCLUSIVE MODE.
    • Both read target/current role inside the transaction.
    • Both count admins inside the same transaction before deleting/demoting an admin.
    • API handlers now call the preserving variants.
    • ErrLastAdmin maps to 403.
    • ErrUserNotFound maps to 404.
    • Invalid roles still map to 400.
    • Self-demotion is still explicitly blocked.

    Tests added:

    • TestPhase2_UpdateUserRolePreservingAdmin_ConcurrentDemotions
    • TestPhase2_DeleteUserPreservingAdmin_ConcurrentDeletes

    Each test creates two admins, runs concurrent destructive admin mutations, expects exactly one success and one auth.ErrLastAdmin, and verifies the final admin count is 1.
