## Bugs Fixed in rc3

### BUG-13: Analyzer Segfault (CRITICAL) -- FIXED

**Root cause:** SPI tuptable clobbering in `sage_upsert_finding()` and `sage_resolve_finding()`. When these functions called `SPI_execute_with_args()`, they overwrote the global `SPI_tuptable` pointer. Caller functions iterating over query results in a for-loop would then read from the wrong tuptable on iteration 2+, causing `pstrdup(NULL)` which triggered SIGSEGV.

**Fix:** Save/restore `SPI_tuptable` and `SPI_processed` around all `SPI_execute_with_args()` calls in `findings.c`. Also fixed a secondary issue: the analyzer's first-boot crash (before schema exists) is now caught by PG_TRY/PG_CATCH.

**Additional fixes in action_executor.c:**
- `sage_rollback_check()`: Fixed type-punning bug (`numeric` returned but `DatumGetFloat8()` used)
- `sage_rollback_check()`: Added missing `SPI_finish()` before early return

**Files:** `src/findings.c`, `src/analyzer.c`, `src/action_executor.c`
**Verification:** 0 SIGSEGV/PANIC across 12+ minutes of continuous operation with 60s analyzer cycles.

### BUG-3: "queries" Snapshot Missing -- FIXED

**Root cause:** PostgreSQL 17 renamed `blk_read_time`/`blk_write_time` to `shared_blk_read_time`/`shared_blk_write_time` in `pg_stat_statements`. The collector query failed silently (caught by PG_CATCH), and "queries" category was never inserted.

**Fix:** Added `#if PG_VERSION_NUM >= 170000` guard to use the correct column names. Matches the existing pattern for `pg_stat_checkpointer` vs `pg_stat_bgwriter`.

**File:** `src/collector.c` (lines 191-195)
**Verification:** "queries" category present with 14 snapshots collected.

### BUG-15: diagnose() Response Truncation -- FIXED

**Root cause:** `json_extract_string()` in `llm.c` had a flawed escape check: `*(pos - 1) != '\\'` doesn't handle double-backslash before quotes (`\\\\"`) correctly, causing early string termination.

**Fix:** Removed the backward-looking escape check from the while condition. The loop body already handles escapes correctly by consuming `\\` sequences forward, so the condition only needs `*pos != '"'`.

**File:** `src/llm.c` (line 129)
**Verification:** `sage.diagnose('What indexes exist?')` returns 6,964 characters (was ~150).

### API Key File Loading -- NEW FEATURE

**Problem:** `sage.llm_api_key` (PGC_SUSET) can't be set via ALTER SYSTEM. Background workers never see session-level SET values.

**Solution:** New GUC `sage.llm_api_key_file` (PGC_SIGHUP) reads the API key from a file on disk. Persists via ALTER SYSTEM, reloads on SIGHUP, supports key rotation.

**Design:**
- `sage_get_llm_api_key()` accessor: session SET > file-loaded > empty
- File read via `AllocateFile()`/`FreeFile()` (PostgreSQL tracked I/O)
- Loaded into `TopMemoryContext` (survives transactions)
- All 3 workers (collector, analyzer, briefing) reload on SIGHUP

**Files:** `src/guc.c`, `src/llm.c`, `src/pg_sage.c`, `src/collector.c`, `src/analyzer.c`, `src/briefing.c`, `include/pg_sage.h`
**Verification:** ALTER SYSTEM + pg_reload_conf() correctly loads key from file.

### Sage Schema Exclusion (Comprehensive) -- FIXED

**Problem:** rc2 excluded `sage` schema from `vacuum_bloat` and `duplicate_index` queries, but not from `unused_index`, `index_bloat`, `index_write_penalty`, `seq_scan`, or `missing_index` queries. Sage's own indexes appeared as findings.

**Fix:** Added `schemaname != 'sage'` / `i.schemaname != 'sage'` to 5 additional queries in `analyzer.c`:
- Unused index detection (line 283)
- Seq scan advisory (line 835)
- Seq scan watchdog (line 1256)
- Index bloat estimation (line 1944)
- Index write penalty (line 2055)

**File:** `src/analyzer.c`
**Verification:** 0 findings with `sage.%` object identifiers.

### BUG-14: analyzer_interval GUC -- NOT A CODE BUG

The `pg_sage_test` container was started with `-c sage.analyzer_interval=3600` on the command line, which overrides ALTER SYSTEM per PostgreSQL precedence rules. The GUC code correctly reads the value each cycle. Fixed operationally by not using `-c` flag.
