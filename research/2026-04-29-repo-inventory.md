## Architectural Constraints

- `pg_stat_statements` is a core dependency for query-level features; without it, slow query, regression, missing-index, tuner, and related analysis degrade or disappear. Evidence: `docs/security.md:47`, `sidecar/internal/startup/checks.go`.
- Optional extensions shape depth: HypoPG improves index validation, `pg_hint_plan` enables persistent hints, and `auto_explain` improves plan capture; the sidecar has fallbacks but weaker validation when these are absent. Evidence: `roadmap.md:116-118`, `docs/architecture.md:60-70`, `docs/architecture.md:92-102`.
- The safety model is trust-ramped and gated by emergency stop, maintenance windows, circuit breakers, replica checks, action risk, and rollback logging. Evidence: `README.md:35`, `docs/security.md:61-76`, `docs/security.md:158-176`, `docs/architecture.md:72-82`.
- Fleet mode uses per-database pools/goroutines while sharing some process-wide resources, notably the ANALYZE semaphore and logwatch fanouts. Evidence: `CLAUDE.md:33-36`, `sidecar/cmd/pg_sage_sidecar/main.go:55-58`, `sidecar/cmd/pg_sage_sidecar/main.go:1121-1215`.
- LLM features are optional, OpenAI-compatible, budgeted, and circuit-broken; config secrets are tagged and expected to come from env/secrets rather than files. Evidence: `sidecar/internal/config/config.go:162-174`, `docs/security.md:92-110`, `docs/security.md:166-172`.
- API/UI are embedded into the same sidecar process and served from `:8080`; there is no separate deployment unit for the dashboard. Evidence: `README.md:25`, `sidecar/web/README.md:1-18`.

## Underdeveloped Areas / Obvious Product Gaps

- Roadmap and public docs are stale relative to the implementation. README says 49 REST endpoints, CLAUDE says 17, while the current API scan sees 75 unique route patterns and code comments reference v0.9/v0.11/v0.12 features. Evidence: `README.md:40`, `CLAUDE.md:130-153`, `sidecar/internal/api/router.go:224-299`.
- Finding docs lag the product. `docs/findings.md` documents the older rule set and LLM index optimizer, but not cases, RCA incidents, lock chains, work_mem promotion, extension drift, schema_lint categories, migration safety, shadow mode, or provider readiness. Evidence: `docs/findings.md:12-347`, `sidecar/internal/analyzer/analyzer.go:372-405`, `sidecar/internal/schema/lint/runner.go:16-25`.
- Config documentation and examples do not expose the newer knobs cleanly. The config struct has RCA, runaway, explain, logwatch, schema_lint, and migration sections, but `docs/configuration.md` only summarizes core/trust/LLM/retention and points to `config.example.yaml`, which is still v0.8.5-focused and omits obvious v0.9/v0.10 sections. Evidence: `sidecar/internal/config/config.go:62-67`, `sidecar/internal/config/config.go:316-358`, `docs/configuration.md:122-182`, `sidecar/config.example.yaml:1-220`.
- Hint revalidation is implemented, but not to the full v0.8.5 spec. The spec calls for cost comparison, EXPLAIN timeout handling, and keep/rollback ratios; current `Revalidate` decides by age, object existence, queryid presence, stagnant calls, and mean execution time. Local scan found the ratio fields in config/types but not active tuner logic. Evidence: `docs/plan_v0.8.5.md:76-88`, `sidecar/internal/tuner/revalidate.go:89-167`, `sidecar/internal/tuner/types.go:67-69`.
- Extension drift is narrower than the spec's required-extension guardrail. The spec wants missing `pg_stat_statements`/`pg_hint_plan` preflight, shared_preload detection, throttled retry, and recovery findings; the current analyzer rule reports installed extension versions that differ from available defaults. Evidence: `docs/plan_v0.8.5.md:551-590`, `sidecar/internal/analyzer/rules_extension_drift.go:8-26`.
- Schema intelligence is partly split between code and unresolved research/spec concerns. The linter has many rules and persists into unified findings, but v0.10 reviews still call out edge cases around temp tables, partitioning, non-public schemas, PgBouncer/N+1, matview refresh tracking, API shapes, and rate limiting. Evidence: `sidecar/internal/schema/lint/linter.go:105-145`, `research/v010-spec-review.md:216-433`, `research/v010-spec-review.md:516-890`.
- Migration safety is advisory-only and activity-polling based. The advisor explicitly falls back to advisory mode when another mode is requested, and log-based DDL detection remains tied to logwatch/log_statement readiness. Evidence: `sidecar/internal/migration/advisor.go:15-17`, `sidecar/internal/migration/advisor.go:29-43`, `sidecar/internal/migration/detector.go:13-49`, `research/v010-spec-review.md:15-37`.
- UI redesign is mid-migration. The code now aliases old routes into a cases-first queue, but several older route concepts/files still exist and docs/design language are inconsistent across README, web README, and UI redesign notes. Evidence: `docs/ui-redesign-v2.md:60-72`, `sidecar/web/src/App.jsx:143-169`, `sidecar/web/README.md:23-36`.
- Time-range plumbing is not universal. The helper exists and is used in Findings/Actions/FleetHealthChart, while the redesign success criteria call for every on-screen chart to rescope with the picker. Evidence: `docs/ui-redesign-v2.md:372`, `docs/ui-redesign-v2.md:535`, `sidecar/web/src/hooks/useAPI.js:71-79`, `sidecar/web/src/pages/Actions.jsx:52-59`, `sidecar/web/src/pages/Findings.jsx:131-152`.
- Integration confidence is environment-sensitive. The repo has large test volume, but local inventory found 146 actual `t.Skip*` calls, mostly gated on live Postgres, extensions, cloud-like behavior, or LLM keys. Evidence: `sidecar/e2e/fleet_test.go:237-238`, `sidecar/e2e/llm_integration_test.go:36-42`, `sidecar/internal/tuner/revalidate_test.go:261`, `sidecar/test/hint_verify/hint_verify_test.go:125-128`.
- Explicitly deferred product areas remain important: parameter-sensitive query detection, CREATE STATISTICS advisor, REINDEX/index-bloat advisor, autonomous ALTER ROLE, LLM-driven revalidation, tooltip tier 3-4 fields, and PG18 support. Evidence: `docs/plan_v0.8.5.md:1327-1337`.
